# Terms of Service Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a `/tos` route with MDX content and a reusable `DocLayout` component styled to match the is-pinoy.dev pixel-art design system.

**Architecture:** `@next/mdx` processes `app/tos/page.mdx` natively in Next.js App Router. A `DocLayout` component in `apps/web/components/doc-layout.tsx` wraps any doc page with the fixed nav, scanline overlay, and doc typography. `app/tos/layout.tsx` applies `DocLayout` to the MDX page. The landing page gets a minimal footer with a ToS link.

**Tech Stack:** Next.js 16 App Router, `@next/mdx`, React 19, inline styles matching DESIGN.md tokens.

---

| `apps/web/app/tos/page.mdx` | Create | Full Terms of Service content |
