Once your subdomain is registered, you can add a badge to your README, GitHub profile, or any webpage to show it off.

All badges are served from `badges.is-pinoy.dev` as SVG, PNG, or WebP.

<SubdomainBanner />

Click any theme to select it and copy the embed code.

## Subdomain Badge

The general-purpose badge. Shows your subdomain with a customizable eyebrow label (max 20 characters, defaults to `DEPLOYED ON`).

<BadgeThemes basePath="/badge/yourname" themes={["light", "dark", "gold", "outlined"]} alt="subdomain badge" />

Custom label via `?label=`:

<BadgeThemes basePath="/badge/yourname?label=BUILT+BY" themes={["light", "dark", "gold", "outlined"]} alt="subdomain badge with custom label" />

## Interactive HTML badge

For your own website, portfolio, or footer — anywhere real HTML runs — use the `<is-pinoy-badge>` web component instead of a static image. On top of the static badge it can add on-brand sun motion: a slow continuous rotation or a springy turn on hover. Motion moves only the mark and switches off under `prefers-reduced-motion`. (GitHub READMEs strip scripts, so keep using the SVG/Markdown embeds there.)

Try it live — pick a type, theme, and motion:

<InteractiveBadge />

Load the component once per page, then drop in as many badges as you like:

```html
<script src="https://badges.is-pinoy.dev/badge.js"></script>

<is-pinoy-badge handle="yourname" type="deployed-on" theme="light"></is-pinoy-badge>
```

It renders into Shadow DOM, so your page's CSS can't bleed in, and it respects `prefers-reduced-motion`.

| Attribute | Values | Default |
|-----------|--------|---------|
| `handle` | your subdomain (omit for `pinoy-made`/`certified`) | — |
| `type` | `deployed-on`, `member`, `pinoy-made`, `certified` | `deployed-on` |
| `theme` | `light`, `dark`, `gold`, `outlined` | per-type default |
| `label` | custom eyebrow for `deployed-on` | `DEPLOYED ON` |
| `icon` | `false` hides the sun mark cell | shown |
| `animate` | `spin` (continuous) or `hover` (on hover) | none |
| `bg` `text` `muted` `border` `mark` `markbg` | color overrides (hex or `transparent`) | theme value |

```html
<is-pinoy-badge handle="yourname" type="certified" theme="gold" shimmer="loop"></is-pinoy-badge>
```

## Not found state

If your subdomain is not registered, the badge shows a greyed-out "not found" state instead of your handle. Register your subdomain first via the [steps](/guides/workflow).
