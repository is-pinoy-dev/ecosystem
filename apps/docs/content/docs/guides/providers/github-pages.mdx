This guide walks you through getting your GitHub Pages site live on an is-pinoy.dev subdomain. Unlike platforms that hand you a unique DNS value, GitHub Pages uses a predictable CNAME target based on your username.

## Prerequisites

- Your portfolio is already deployed and building on GitHub Pages
- You have chosen a subdomain (e.g. `yourname.is-pinoy.dev`)

---

<Steps>
  <Step>
    ## Find your CNAME value

    For a **user or organization site** (the repo named `username.github.io`), your CNAME target is:

    ```
    username.github.io.
    ```

    For a **project site** (any other repo, served at `username.github.io/repo`), the CNAME target is the same — you still point at `username.github.io.`, and GitHub routes to the correct repo using the custom domain you set in the next step.

    <Callout type="warn">
      Keep the trailing dot (`.`). This is standard DNS notation for a fully-qualified domain name. Replace `username` with your own GitHub username, lowercased.
    </Callout>
  </Step>
  <Step>
    ## Set the custom domain in your repo

    1. Go to your repository on GitHub
    2. Click **Settings** → **Pages**
    3. Under **Custom domain**, enter your full subdomain:

    ```
    yourname.is-pinoy.dev
    ```

    4. Click **Save**

    This commits a `CNAME` file to your repository so GitHub Pages knows to serve your site at that hostname.

    <Callout type="info">
      GitHub will report "DNS check unsuccessful" until your is-pinoy.dev PR is merged and DNS has propagated. This is expected — GitHub cannot see the record until it is live.
    </Callout>
  </Step>
  <Step>
    ## Choose your Cloudflare proxy setting

    is-pinoy.dev manages DNS through Cloudflare. You can choose whether traffic is **proxied** through Cloudflare's network (the orange cloud) using the `proxied` field on your record.

    <Callout type="warn">
      **Leave the proxy off (`proxied: false`, the default) until HTTPS is working.** GitHub Pages provisions a Let's Encrypt certificate by validating your domain over plain DNS. If Cloudflare proxies the record before the certificate is issued, validation can fail and **Enforce HTTPS** will stay greyed out in your Pages settings.
    </Callout>

    Once your site loads over HTTPS and **Enforce HTTPS** is enabled in GitHub Pages settings, you may optionally switch `proxied` to `true` to get Cloudflare's CDN caching and DDoS protection. See [Cloudflare proxy](/guides/reference/supported-records#cloudflare-proxy-proxied) for details.
  </Step>
  <Step>
    ## Create your subdomain JSON file

    Now that you have your CNAME value, go to [Create Your JSON File](/guides/workflow/create-file) to add it to your file.

    For reference, here is what your JSON should look like:

    <Tabs items={["DNS-only (recommended)", "Proxied through Cloudflare"]}>
      <Tab value="DNS-only (recommended)">
        ```json
        {
          "$schema": "https://raw.githubusercontent.com/is-pinoy-dev/ecosystem/main/packages/schemas/schema/v1/subdomain.schema.json",
          "subdomain": "yourname",
          "owner": {
            "github": "your-github-username"
          },
          "records": {
            "CNAME": {
              "value": "your-github-username.github.io."
            }
          }
        }
        ```
      </Tab>
      <Tab value="Proxied through Cloudflare">
        ```json
        {
          "$schema": "https://raw.githubusercontent.com/is-pinoy-dev/ecosystem/main/packages/schemas/schema/v1/subdomain.schema.json",
          "subdomain": "yourname",
          "owner": {
            "github": "your-github-username"
          },
          "records": {
            "CNAME": {
              "value": "your-github-username.github.io.",
              "proxied": true
            }
          }
        }
        ```
      </Tab>
    </Tabs>

    Replace `yourname` and `your-github-username` with your actual information.
  </Step>
</Steps>

## Important notes

- **Trailing dot** — the CNAME value must end with `.` (e.g. `username.github.io.`)
- **No TXT record needed** — serving a GitHub Pages site over a subdomain only requires the CNAME. GitHub's organization-level domain verification uses a separate `_github-pages-challenge` host that is not part of this single-subdomain file, so skip it.
- **HTTPS first, proxy second** — wait until GitHub Pages has issued your certificate and **Enforce HTTPS** is on before enabling `proxied: true`
- **"DNS check unsuccessful" on GitHub** — expected until your PR is merged and DNS propagates
